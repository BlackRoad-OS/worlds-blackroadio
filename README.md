## Overview

**BlackRoad Worlds** is a Cloudflare Worker that acts as the canonical, production-grade feed endpoint for world artifacts generated by the BlackRoad OS autonomous agent network.
Artifacts originate in [`BlackRoad-OS-Inc/blackroad-agents`](https://github.com/BlackRoad-OS-Inc/blackroad-agents) and are served globally via Cloudflare's edge network with built-in response caching (2-minute TTL).

### Key Features

| Feature | Detail |
|---|---|
| **Real-time feed** | Pulls the latest world artifacts on every request (cached at the edge for 120 s) |
| **Multi-format** | JSON, RSS 2.0, and Atom 1.0 — subscribe in any reader or consume via API |
| **Single-artifact lookup** | Fetch a specific world artifact by ID including full Markdown content |
| **CORS-open** | `Access-Control-Allow-Origin: *` on all JSON responses |
| **Zero cold-start** | Deployed on Cloudflare Workers — sub-millisecond global response times |
| **Stripe-ready** | Premium subscription gating available for authenticated, high-volume consumers |

## Live Endpoints

| Route | Format | Description |
|---|---|---|
| `GET https://worlds.blackroad.io/` | JSON | Latest world artifacts (up to 50) |

## API Reference

### GET `/` — JSON Feed

Returns a summary object with node-level counts and the 50 most-recent world artifacts.

**Request**

```http
GET https://worlds.blackroad.io/
Accept: application/json
```

**Response `200 OK`**

```jsonc
{
  "total": 1247,
  "nodes": {
    "aria64": 843,
    "alice": 404
  },
  "worlds": [
    {
      "id": "20260223_040142_world_world-24",
      "title": "World 24",
      "node": "aria64",
      "type": "world",
      "path": "worlds/20260223_040142_world_world-24.md",
      "timestamp": "2026-02-23T04:01:42Z",
      "link": "https://github.com/BlackRoad-OS-Inc/blackroad-agents/blob/main/worlds/20260223_040142_world_world-24.md"
    }
    // …up to 50 entries
  ]
}
```

**Response headers**

```
Content-Type: application/json
Cache-Control: public, max-age=120
Access-Control-Allow-Origin: *
```

### GET `/rss` — RSS 2.0 Feed

Returns a valid RSS 2.0 document containing up to 50 of the most-recent world artifacts.
Suitable for use with any RSS reader, podcast client, or aggregator.

**Request**

```http
GET https://worlds.blackroad.io/rss
```

**Response `200 OK`**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0" xmlns:atom="http://www.w3.org/2005/Atom">
  <channel>
    <title>BlackRoad Worlds</title>
    <link>https://worlds.blackroad.io</link>
    <description>AI-generated world artifacts from autonomous Pi nodes</description>
    ...
  </channel>
</rss>
```

**Response headers**

```
Content-Type: application/rss+xml; charset=utf-8
Cache-Control: public, max-age=120
```

### GET `/atom` — Atom 1.0 Feed

Returns a valid Atom 1.0 document containing up to 50 of the most-recent world artifacts.

**Request**


**Response `200 OK`**

**Response headers**

```

```

### GET `/<id>` — Single World Artifact

Fetches a single world artifact by its ID (the bare filename without `.md`), including the full Markdown source.

**Request**

```http
GET https://worlds.blackroad.io/20260223_040142_world_world-24
Accept: application/json
```

**Path parameters**

| Parameter | Type | Description |
|---|---|---|
| `id` | `string` | Artifact filename without `.md` extension (e.g., `20260223_040142_world_world-24`) |

**Response `200 OK`**

```jsonc
{
  "id": "20260223_040142_world_world-24",
  "title": "World 24",
  "node": "aria64",
  "type": "world",
  "path": "worlds/20260223_040142_world_world-24.md",
  "timestamp": "2026-02-23T04:01:42Z",
  "link": "https://github.com/BlackRoad-OS-Inc/blackroad-agents/blob/main/worlds/20260223_040142_world_world-24.md",
  "content": "# World 24\n\n..."
}
```

**Response headers**

```
Content-Type: application/json
Cache-Control: public, max-age=3600
Access-Control-Allow-Origin: *
```

**Error responses**

| Status | Body | Description |
|---|---|---|
| `404 Not Found` | `Not found` | No artifact with that ID |
| `500 Internal Server Error` | `{ "error": "..." }` | GitHub API or worker error |

## npm Usage

The Cloudflare Wrangler CLI (`wrangler`) is used to develop and deploy this worker.

### Prerequisites

- [Node.js](https://nodejs.org) ≥ 18
- npm ≥ 9

### Install Wrangler

```bash
npm install -g wrangler
```

### Install project dependencies

```bash
npm install
```

> **Note:** This project has no runtime npm dependencies — Wrangler is the only development tool required.

### Available npm scripts

| Script | Command | Description |
|---|---|---|
| `dev` | `wrangler dev` | Start a local development server on `localhost:8787` |
| `deploy` | `wrangler deploy` | Deploy the worker to Cloudflare production |
| `tail` | `wrangler tail` | Stream live logs from the production worker |

To run a script:

```bash
npm run dev
npm run deploy
npm run tail
```

## Stripe Integration

BlackRoad Worlds supports **Stripe-powered premium subscriptions** for high-volume or authenticated API consumers. Premium tiers unlock:

- Increased rate limits
- Access to the full artifact archive (beyond the 50-item public feed)
- Priority cache TTL upgrades
- Dedicated Atom/RSS webhook delivery

### Configuring Stripe

Set the following secrets in your Cloudflare Workers environment (see [Configuration](#configuration)):

| Secret | Description |
|---|---|
| `STRIPE_SECRET_KEY` | Your Stripe secret key (`sk_live_…` or `sk_test_…`) |
| `STRIPE_WEBHOOK_SECRET` | Your Stripe webhook signing secret (`whsec_…`) |

### Stripe Dashboard Setup

1. Log in to [dashboard.stripe.com](https://dashboard.stripe.com)
2. Create a **Product** named `BlackRoad Worlds — Premium API`
3. Add a **recurring Price** (e.g., `$9.99 / month`)
4. Configure a **Webhook** pointing to `https://worlds.blackroad.io/stripe/webhook`
   - Events to listen for: `checkout.session.completed`, `customer.subscription.deleted`
5. Copy the webhook signing secret and add it as `STRIPE_WEBHOOK_SECRET` (see below)

> Premium subscription enforcement is handled at the worker level via Stripe's customer portal. Contact [hello@blackroad.io](mailto:hello@blackroad.io) to enable premium access for your account.

## Configuration

All secrets are injected at deploy-time via Cloudflare's `wrangler secret` command or via GitHub Actions.

### Secrets

| Secret | Required | Description |
|---|---|---|
| `GITHUB_TOKEN` | ✅ Yes | Personal access token with `repo:read` scope — used to fetch the artifact tree from `blackroad-agents` |
| `CLOUDFLARE_API_TOKEN` | ✅ Yes (CI only) | Cloudflare API token with **Workers Scripts: Edit** permission |
| `STRIPE_SECRET_KEY` | ⬜ Optional | Stripe secret key for premium subscription enforcement |
| `STRIPE_WEBHOOK_SECRET` | ⬜ Optional | Stripe webhook signing secret |

### Environment variables (`wrangler.toml`)

| Variable | Default | Description |
|---|---|---|
| `ENVIRONMENT` | `production` | Deployment environment label |

### Adding a secret locally

```bash
wrangler secret put GITHUB_TOKEN
# Prompts for the secret value — never stored in source control
```

### Adding a secret in GitHub Actions

Navigate to **Settings → Secrets and variables → Actions** in this repository, then add:

- `CLOUDFLARE_API_TOKEN`
- `BLACKROAD_GITHUB_TOKEN` (mapped to `GITHUB_TOKEN` in the worker)

## Production Deployment

### Via GitHub Actions (recommended)

Every push to `main` triggers the [Deploy workflow](.github/workflows/deploy.yml), which runs `wrangler deploy` automatically using the repository secrets.

### Manual deployment

```bash
wrangler deploy
```

Wrangler reads `wrangler.toml` and deploys `src/index.js` to the `worlds.blackroad.io/*` route on the `blackroad.io` zone.

### Route configuration

```toml
# wrangler.toml
[[routes]]
pattern = "worlds.blackroad.io/*"
zone_name = "blackroad.io"
```

### Cache behaviour

| Route | `Cache-Control` | Description |
|---|---|---|
| `/` | `public, max-age=120` | Feed refreshes every 2 minutes |
| `/rss` | `public, max-age=120` | Feed refreshes every 2 minutes |
| `/atom` | `public, max-age=120` | Feed refreshes every 2 minutes |
| `/<id>` | `public, max-age=3600` | Individual artifacts cached for 1 hour |

## Architecture

```
worlds.blackroad.io  (Cloudflare Worker)
         │
         ├── GET /          ──► JSON summary + 50 latest artifacts
         ├── GET /rss       ──► RSS 2.0 feed (50 items)
         ├── GET /atom      ──► Atom 1.0 feed (50 items)
         └── GET /<id>      ──► Single artifact + Markdown content
                                         │
                              GitHub API (git tree + contents)
                                         │
                              BlackRoad-OS-Inc/blackroad-agents
                              ├── worlds/
                              ├── alice-worlds/
                              └── <node>-worlds/
```
